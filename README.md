# Made in Rwanda Content Recommender (Tier 1)

## Project Overview
This project is a simple content-based recommender for a local Made-in-Rwanda artisan catalog.  
Given a buyer query (English, French, or mixed), it ranks local products and returns the top matches.

Scope is intentionally small and defendable for a Tier 1 hackathon:
- one local catalog
- TF-IDF text matching
- lightweight local-boost fallback using click popularity
- clear offline evaluation in `eval.ipynb`

## Why TF-IDF Instead of Embeddings
TF-IDF was chosen because it is:
- easy to explain live (token overlap and weighted terms)
- fast to train and query on small datasets
- reproducible and offline (no API, no paid services)
- aligned with Tier 1 constraints and available time

Embeddings can improve semantic recall, but add complexity and are not required for this baseline stage.

## Project Structure
- `generate_data.py`: deterministic synthetic data generator aligned to the brief
- `recommender.py`: CLI recommender (TF-IDF + cosine + simple local-boost fallback)
- `inspect_data.py`: quick dataset inspection utility
- `eval.ipynb`: minimal offline evaluation notebook
- `data/catalog.csv`: local substitute catalog
- `data/queries.csv`: buyer intent queries + synthetic evaluation anchor
- `data/click_log.csv`: synthetic click events for popularity signal

## Quick Start (2 Commands)
```bash
pip install -r requirements.txt
python generate_data.py && python recommender.py --q "leather gift for women" && python -m jupyter nbconvert --to notebook --execute eval.ipynb --inplace
```
French equivalent used in the brief: `"cadeau en cuir pour femme"` (English: leather gift for women).

## Local-Boost Rule (Brief)
Main ranking is always TF-IDF + cosine similarity over the local catalog.

Fallback is used only when the top cosine score is below threshold:
1. prefer items whose `category` or `material` matches obvious query keywords
2. break ties using click-log popularity (`click_log.csv`)
3. keep cosine score as final tie-break

This keeps behavior simple, local-first, and easy to justify.

## Evaluation Metrics
`eval.ipynb` reports:
- `NDCG@5` using the synthetic query reference SKU as relevance anchor
- `Local-presence rate @3` (share of queries with at least one local item in top 3)
- average number of returned items

## Current Results
| Model | Mean NDCG@5 | Local-Presence Rate @3 | Avg Items Returned |
|---|---:|---:|---:|
| TF-IDF + Local-Boost Fallback | `0.3413` | `1.0000` | `5.00` |

## Limitations
- Data is synthetic, so scores are useful for sanity checks, not final market performance claims.
- Relevance anchor is a single synthetic best-local SKU per query, which simplifies real buyer preference.
- TF-IDF relies on token overlap and can miss semantic matches when wording differs a lot.

## Defense Notes
- Why TF-IDF: simple, fast, offline, and easy to explain in live defense under Tier 1 constraints.
- Query type where TF-IDF may fail: intent-heavy semantic queries with little lexical overlap (for example, abstract style requests or paraphrases).
- How local-boost/fallback works: if top cosine score is strong, return normal TF-IDF top-k; if weak, prefer category/material keyword matches, break ties with click popularity, then use cosine score as final tie-break.

## Data Note
The dataset is synthetic and generated by a reproducible script (`generate_data.py`) to stay aligned with the hackathon brief while keeping the pipeline fully offline and explainable.
